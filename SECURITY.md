# Security Policy

## Supported Versions

This project is still in active development and does not yet have a formal supported-version policy.
Security fixes are expected to land on the main development line first.

## Reporting a Vulnerability

Please do not disclose security issues publicly before they are reviewed.

If GitHub Private Vulnerability Reporting is enabled for this repository, please use the
`Security` tab to submit a private report. Otherwise, contact the maintainers through the
private channel documented in the repository profile or organization settings before opening
any public issue.

When reporting a vulnerability, include:

- a clear description of the issue
- impact assessment
- reproduction steps
- affected files or modules
- suggested remediation if available

If a private reporting channel is added later, this document should be updated accordingly.

## Scope

Security issues may include:

- unauthorized access to user data
- privilege escalation
- insecure database exposure
- credential leakage
- unsafe file or storage handling
